# Claude Code 安装教程

> **环境说明**：Windows 系统

---

## 一、安装步骤（Windows）

### 第一步：运行安装脚本

打开 **PowerShell**，确保代理已开启，执行：

```powershell
irm https://claude.ai/install.ps1 | iex
```

脚本会自动下载并安装 Claude Code CLI。

### 第二步：验证安装

```powershell
claude doctor
```

**若出现以下报错**，说明安装路径未加入环境变量：

```
无法将"claude"项识别为 cmdlet、函数、脚本文件或可运行程序的名称。
请检查名称的拼写，如果包括路径，请确保路径正确，然后再试一次。
```

**解决方法**：手动将安装目录加入 PATH：

```powershell
[Environment]::SetEnvironmentVariable("PATH", $env:PATH + ";C:\Users\Administrator\.local\bin", "User")
```

> 执行后**重新打开** PowerShell 窗口使环境变量生效。

### 第三步：再次验证

```powershell
claude doctor
```

看到各项检查通过后，执行：

```powershell
claude
```

出现交互界面即表示**安装成功**。

## 二、国内使用前置条件

Claude Code 的注册和使用对中国大陆用户有一定门槛，安装前需要准备好以下条件：

### 1. 科学上网（代理软件）

Claude 的所有服务（官网、API、Claude Code）均无法在大陆直连访问，必须全程开启代理。

- 推荐使用支持**规则分流**的代理客户端（如 Clash Verge、Mihomo Party 等）
- 代理需要设置为**全局模式**或确保 `claude.ai`、`anthropic.com` 走代理节点
- 终端（PowerShell / CMD）默认不走系统代理，需要额外配置
- 在代理软件中开启 **TUN 模式**，让所有流量（包括终端）自动走代理，无需手动配置
- 使用**软路由**在网关层面接管全局流量，终端无需任何额外配置
- 使用**机场节点 + 链式代理**的方式，将流量经过链式代理，获得更稳定的住宅出口 IP

### 2. 稳定的住宅 IP 节点

Claude 对 IP 风控较严，以下情况容易触发封号或验证：

| IP 类型 | 风险等级 | 说明 |
|---------|:-------:|------|
| 数据中心 IP（VPS/机房） | 🔴 高 | 极易被识别拦截，注册和使用均可能失败 |
| 共享住宅 IP | 🟡 中 | 多人共用同一 IP，存在风险 |
| 独享住宅 IP | 🟢 低 | 最稳定，推荐用于注册和长期使用 |

**建议**：注册账号时使用固定的住宅 IP 节点，后续使用也尽量保持同一节点，避免频繁切换地区。

### 3. 国外手机号

Claude 注册需要**手机号验证**，目前不支持中国大陆（+86）和香港（+852）号码。

**可用方案**：

- **实体 SIM 卡 / eSIM**：美国、英国、日本等国家的号码，稳定性最高。eSIM 需要手机支持 eSIM 功能，且所购买的号商支持远程下载配置文件，部分 eSIM 号商不提供下载功能，购买前注意确认。
- **虚拟号码平台（接码平台）**：可以用，按次付费，接码平台号码为一次性使用，**不建议用于长期账号绑定**

> **注意**：接码平台的号码复用率高，部分号码已被 Anthropic 标记，可能导致验证失败。如果条件允许，优先使用实体 SIM 卡或 eSIM。大家八仙过海，各显神通，找到适合自己的方式就好。

## 三、常见问题

**Q：安装脚本执行超时或失败？**

检查代理是否对终端生效，参考第一节设置 `HTTPS_PROXY` 环境变量后重试。

**Q：`claude doctor` 提示 Node.js 版本不符？**

Claude Code 要求 Node.js 18+，前往 [nodejs.org](https://nodejs.org) 下载最新 LTS 版本。

**Q：登录时提示地区不支持？**

切换到住宅 IP 节点再尝试登录，数据中心 IP 容易被拒绝。

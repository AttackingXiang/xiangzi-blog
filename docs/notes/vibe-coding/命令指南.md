# Claude Code 命令指南

> Claude Code CLI 斜杠命令（Slash Commands）按使用频次排列，含详细解释与示例。

---

## 最高频命令

### `/help`
显示所有可用命令和帮助信息。

- **用途**：新会话定向、发现可用功能
- **示例**：直接输入 `/help` 查看当前环境下所有命令

---

### `/clear`（别名：`/reset`、`/new`）
清空对话历史，释放上下文窗口空间，开启新对话。

- **用途**：上下文占满时重置、在同一会话中开始新话题
- **示例**：`/clear`
- **注意**：保留 Session ID，可通过 `/resume` 恢复

---

### `/config`（别名：`/settings`）
打开设置界面，调整主题、模型、输出风格等偏好。

- **用途**：换模型、改外观、调整输出格式
- **示例**：`/config` 打开交互式设置菜单

---

### `/model [模型名]`
选择或切换 AI 模型。

- **用途**：在 Sonnet / Opus 等不同模型之间切换，权衡速度和能力
- **示例**：`/model opus`、`/model sonnet`、`/model claude-opus-4-6`
- **注意**：立即生效，支持箭头键调整 effort 等级

---

### `/compact [指令]`
压缩对话历史，可附加保留重点的说明，节省上下文。

- **用途**：上下文不足时减少 Token 占用，同时保留关键信息
- **示例**：`/compact`、`/compact "保留认证逻辑相关内容"`

---

### `/cost`
查看当前会话的 Token 用量和费用统计。

- **用途**：监控 API 开销、管理预算
- **示例**：`/cost` 显示本次会话累计消耗

---

## 高频工作流命令

### `/resume [会话ID或名称]`（别名：`/continue`）
恢复之前的对话。

- **用途**：接续上次未完成的工作
- **示例**：`/resume auth-refactor`、`/resume`（打开选择列表）、`claude --resume abc123`（CLI 启动时）

---

### `/rewind`（别名：`/checkpoint`）
回滚对话和代码到某个历史节点。

- **用途**：撤销不想要的改动、回到稳定状态、创建对话分支
- **示例**：`/rewind` 打开检查点菜单，或使用快捷键 `Esc+Esc`

---

### `/diff`
打开交互式 diff 查看器，展示未提交的改动及每轮对话的变更。

- **用途**：Review Claude 做的代码改动、对比各轮变化
- **示例**：`/diff`，左右箭头切换 git diff / 单轮 diff，上下箭头浏览文件

---

### `/plan [描述]`
进入规划模式，先分析再执行，不直接修改代码。

- **用途**：在动手前审视方案、安全探索
- **示例**：`/plan 修复认证 bug`

---

### `/init`
在项目根目录初始化 `CLAUDE.md` 指导文件。

- **用途**：首次设置项目上下文和持久指令
- **示例**：`/init` 生成 CLAUDE.md 模板

---

### `/memory`
编辑 `CLAUDE.md` 记忆文件，管理自动记忆条目。

- **用途**：维护跨会话的项目级指令
- **示例**：`/memory` 打开记忆编辑器

---

### `/rename [名称]`
为当前会话命名，便于后续识别和恢复。

- **用途**：给会话贴标签，方便组织管理
- **示例**：`/rename auth-refactor`、`/rename`（自动从历史生成名称）

---

## 代码质量命令

### `/security-review`
扫描当前分支的待提交变更，检查安全漏洞。

- **用途**：提交前的安全审查
- **检测范围**：注入漏洞、认证问题、数据暴露等

---

### `/pr-comments [PR编号]`
从 GitHub PR 拉取并展示评论。

- **用途**：查看 PR 反馈、了解讨论内容
- **示例**：`/pr-comments 123`、`/pr-comments https://github.com/owner/repo/pull/123`
- **前提**：已安装并认证 `gh` CLI

---

### `/simplify`（内置 Skill）
Review 已改动代码，检查复用性、质量和效率，发现问题后修复。

- **用途**：代码优化、质量把关

---

## 上下文管理命令

### `/context`
以彩色网格可视化当前上下文用量，并给出优化建议。

- **用途**：了解 Token 消耗、定位优化机会
- **示例**：`/context` 显示详细的上下文分布

---

### `/effort [low|medium|high|max|auto]`
设置模型的思考深度等级。

- **用途**：用速度换精度，或为复杂问题分配更多算力
- **示例**：`/effort high`、`/effort low`、`/effort max`（仅 Opus 4.6）
- **注意**：`low/medium/high` 持久生效，`max` 仅当前会话有效

---

### `/copy [N]`
将最近一条（或倒数第 N 条）助手回复复制到剪贴板。

- **用途**：提取代码或文本
- **示例**：`/copy`、`/copy 2`（倒数第二条）
- **进阶**：按 `w` 可直接写入文件

---

## 配置与外观命令

### `/theme`
更改颜色主题（亮色、暗色、色盲友好、ANSI）。

- **示例**：`/theme` 打开主题选择器

---

### `/vim`
切换 Vim / 普通编辑模式。

- **用途**：使用 Vim 键位操作输入框
- **示例**：`/vim` 开关 Vim 模式

---

### `/keybindings`
打开或创建键位配置文件 `~/.claude/keybindings.json`。

- **用途**：自定义快捷键
- **示例**：`/keybindings`

---

### `/terminal-setup`
配置终端的 Shift+Enter 等快捷键映射。

- **用途**：在 VS Code、Alacritty、Warp 等终端补全缺失的按键绑定

---

### `/color [颜色|default]`
设置当前会话的提示栏颜色，方便区分不同会话。

- **示例**：`/color blue`、`/color red`、`/color default`
- **可用颜色**：red、blue、green、yellow、purple、orange、pink、cyan

---

### `/statusline`
配置状态栏 UI，显示上下文用量、git 状态、费用等信息。

---

## 集成与平台命令

### `/ide`
管理 IDE 集成状态（VS Code、JetBrains 等）。

- **示例**：`/ide` 打开 IDE 管理界面

---

### `/mcp`
管理 Model Context Protocol 服务器连接和 OAuth 认证。

- **用途**：配置外部工具和数据源
- **示例**：`/mcp` 打开 MCP 管理界面

---

### `/desktop`（别名：`/app`）
将当前会话迁移到 Claude Code 桌面应用继续。

- **平台**：仅 macOS 和 Windows

---

### `/chrome`
配置 Claude 的 Chrome 浏览器集成设置。

- **用途**：设置 Web 自动化、浏览器测试

---

### `/hooks`
查看已配置的工具事件钩子（Hooks）。

- **用途**：了解自动化触发器、调试 Hook 行为

---

### `/remote-control`（别名：`/rc`）
让当前会话可从 claude.ai Web 端远程控制。

---

## 插件命令

### `/plugin`
管理 Claude Code 插件（安装、启用、禁用、更新）。

- **示例**：`/plugin install code-review@claude-plugins-official`

---

### `/reload-plugins`
重新加载所有插件，无需重启即可应用更新。

- **用途**：开发插件时热重载

---

### `/skills`
列出所有可用的自定义命令（Skills）。

---

### `/agents`
管理子 Agent 配置。

---

## 调试与分析命令

### `/doctor`
诊断并验证 Claude Code 安装和配置是否正常。

- **用途**：排查问题、验证环境

---

### `/insights`
生成会话分析报告，分析使用模式、摩擦点、生产力指标。

---

### `/stats`
可视化每日用量、会话历史、连续使用天数、模型偏好。

---

### `/debug`（内置 Skill）
为当前会话开启 Debug 日志。

- **用途**：排查问题、观察内部行为

---

### `/status`
打开 Settings 界面，查看版本、模型、账户、网络连通性。

- **注意**：Claude 响应过程中也可使用

---

## 账户与认证命令

### `/login`
登录 Anthropic 账户，启动认证流程。

---

### `/logout`
退出当前 Anthropic 账户。

---

### `/usage`
查看套餐用量上限和当前速率限制状态。

---

### `/upgrade`

---

### `/privacy-settings`

### `/extra-usage`

## 实用工具命令

### `/export [文件名]`
将当前对话导出为纯文本文件。

- **示例**：`/export`、`/export my-conversation.txt`

---

### `/btw <问题>`
提一个不计入对话历史的临时问题，快速获取答案。

- **用途**：快速确认细节、问侧面小问题，不影响主线上下文
- **示例**：`/btw 那个配置文件叫什么名字？`
- **特点**：低费用（复用缓存）、无工具访问、单次响应、不留历史

---

### `/add-dir <路径>`
向当前会话添加新的工作目录，扩大操作范围。

- **示例**：`/add-dir ../other-project`

---

### `/tasks`
列出并管理后台任务。

---

### `/branch [名称]`（别名：`/fork`）
在当前对话节点创建对话分支，探索不同方向。

- **示例**：`/branch alternative-approach`

---

### `/exit`（别名：`/quit`）
退出 Claude Code 会话。

---

## 自动化与调度命令

### `/schedule [描述]`
创建、更新、列出或运行云端定时任务。

- **用途**：设置周期性自动化、调度远程工作
- **示例**：`/schedule 每天早上 9 点跑测试`

---

### `/loop`（内置 Skill）
按固定时间间隔重复执行某条指令或命令。

- **示例**：`/loop 5m /check-status`、`/loop 10m npm test`

---

## 反馈与其他命令

### `/feedback`（别名：`/bug`）
提交对 Claude Code 的反馈或 Bug 报告。

- **示例**：`/feedback`、`/feedback "描述 Bug"`

---

### `/release-notes`
查看完整更新日志，最新版本优先显示。

---

### `/install-github-app`
为仓库设置 Claude GitHub Actions 集成。

---

### `/install-slack-app`
安装 Claude Slack 应用，在 Slack 中使用 Claude。

---

### `/passes`
将一周免费 Claude Code 体验分享给朋友（仅符合条件的账户可见）。

---

### `/stickers`
订购 Claude Code 周边贴纸。

---

## 快速参考

| 场景 | 推荐命令 |
|------|---------|
| 上下文快满了 | `/compact` 或 `/clear` |
| 想换个模型 | `/model opus` / `/model sonnet` |
| 接续昨天的工作 | `/resume` |
| 回滚一次错误修改 | `/rewind` |
| 查看 Claude 改了什么 | `/diff` |
| 先规划再动手 | `/plan` |
| 快速小问题不影响主线 | `/btw` |
| 查费用 | `/cost` |
| 安全审查 | `/security-review` |
| 排查安装问题 | `/doctor` |

---

> **提示**：输入 `/` 后按 Tab 可自动补全命令名称。不同套餐（Free/Pro/Max/Teams）和平台（CLI/Desktop/Web）下可用命令略有差异。

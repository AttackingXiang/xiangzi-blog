##  一、问题现象

### 1️⃣ IPv6 正常

```
ping www.baidu.com
→ 返回 IPv6 地址，正常通
```

------

### 2️⃣ IPv4 异常

```
ping -4 www.baidu.com
→ Network unreachable
```

表现为：

- IPv4 完全不可用
- 但 IPv6 正常访问

------

### 3️⃣ 路由异常

```
ip route
```

结果：

```
192.168.1.0/24 dev eth0 src 192.168.1.7
192.168.10.0/24 dev br-lan src 192.168.10.1
```

❌ 缺失关键项：

```
default via 192.168.1.1
```

------

## 🚨 二、初步误判过程

### ❌ 怀疑 sing-box 导致

检查：

```
ps | grep sing-box
```

结果：

- sing-box 已停止
- 无残留进程

结论：

> ❌ sing-box 不是问题源

------

## 🧠 三、关键问题定位

### 🔥 根本原因

> IPv4 默认路由（default gateway）丢失

------

### 📌 关键证据

- 有 IP：`192.168.1.7`
- ❌ 无 gateway
- ❌ 无 default route

------

## 🧪 四、问题本质解释

### ✔ IPv6 正常原因

IPv6 使用：

- RA（路由通告）
- 自动生成默认路由

所以不依赖 `ip route default`

------

### ❌ IPv4 失败原因

IPv4 必须依赖：

```
default via 192.168.1.1 dev eth0
```

但当前缺失 → 导致：

```
Network unreachable
```

------

## 🧭 五、排查步骤流程

### 1️⃣ 检查进程

```
ps | grep sing-box
```

✔ 排除代理影响

------

### 2️⃣ 检查路由表

```
ip route
```

发现：

- 无 default route

------

### 3️⃣ 检查 WAN 状态（隐含）

```
eth0 = 192.168.1.7
```

说明：

- DHCP 有响应
- 但 gateway 未写入系统

------

## 🛠 六、解决方案

### ✔ 方法1：重启 WAN

```
ifdown wan
ifup wan
```

------

### ✔ 方法2：重新获取 DHCP

```
udhcpc -i eth0
```

------

### ✔ 方法3：重启网络服务（最终修复）

```
/etc/init.d/network restart
```

------

## ✅ 七、修复结果验证

```
ip route
```

应恢复：

```
default via 192.168.1.1 dev eth0
```

------

### 测试 IPv4：

```
ping -4 8.8.8.8
```

✔ 正常返回

------

## 📌 八、最终结论

### 🔥 根因总结

> IPv4 默认路由丢失（DHCP gateway 未写入 / network service 异常）

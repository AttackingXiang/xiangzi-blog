# VPS 基础安全与 UFW 防火墙配置手册

本文档旨在指导如何通过 `ufw` 快速构建服务器安全屏障，涵盖基础加固、节点放行及 Cloudflare 进阶防护。

## 1. 基础防护初始化

在执行任何操作前，**必须首先允许 SSH 端口**，否则启用防火墙后您将无法远程连接服务器。

### 允许核心服务与启用

Bash

```
# 1. 允许 SSH 访问（核心！防止自己被关在外面）
ufw allow 22

# 2. 允许 HTTPS 访问（让 Nginx 的 443 端口对外）
ufw allow 443

# 3. 设置默认拒绝所有入站流量，允许所有出站
ufw default deny incoming
ufw default allow outgoing

# 4. 启用防火墙
ufw enable
```

------

## 2. 业务端口放行

如果您运行了代理服务或自定义节点，需要手动开放对应的入站端口。

### 允许特定节点/端口

Bash

```
# 允许通过的节点：将下方的“入站节点”替换为你的实际端口号
ufw allow 入站节点1
ufw allow 入站节点2

# 重新加载防火墙使规则生效
sudo ufw reload
```

------

## 3. 维护与规则清理

定期检查防火墙状态并清理不再使用的过期端口。

### 查看状态

Bash

```
# 查看防火墙详细运行状态
sudo ufw status verbose
```

### 准确删除不需要的端口

建议使用“编号删除法”，避免输入错误的端口或协议导致误删。

Bash

```
# 1. 查看带编号的规则列表
ufw status numbered

# 执行后，找到对应端口（如 29568/tcp）左侧方括号内的数字编号。

# 2. 执行删除命令（假设编号为 20，请务必以你实际看到的为准）
ufw delete [编号]
```

------

### 💡 核心要点总结

- **SSH 优先级最高**：永远先 `allow 22` 再 `enable`。
- **先删后加**：在配置特定 IP 白名单（如 CF）前，记得先删除原有的全局放行规则。
- **规则顺序**：UFW 是按列表顺序匹配的，编号越小优先级越高。

# ImmortalWrt 旁路由保姆级设置教程

> 适用场景：已有主路由（光猫/运营商路由），将 ImmortalWrt 设备作为旁路由，实现透明代理、广告过滤等功能，不改变主路由拓扑。

---

## 一、基本概念

### 什么是旁路由？

```
[互联网]
    │
[主路由] ←→ [旁路由 (ImmortalWrt)]
    │
[局域网设备]
```

旁路由**不承担主要路由职责**，只作为网关挂在主路由下，客户端将默认网关指向旁路由，流量经旁路由处理后再转发给主路由上网。

### 旁路由 vs 主路由模式对比

| 对比项 | 主路由模式 | 旁路由模式 |
|--------|-----------|-----------|
| 网络拓扑改动 | 大 | 小 |
| 配置复杂度 | 高 | 低 |
| 故障影响范围 | 全网断网 | 仅代理失效 |
| 适合场景 | 全面替换 | 旁挂透明代理 |

## 二、准备工作

### 硬件要求

- 一台刷好 ImmortalWrt 的设备（软路由、旧路由器等）
- 网线或 Wi-Fi 连接到主路由
- 管理电脑

### 固件获取

前往 ImmortalWrt 官方下载页面，根据设备型号选择对应固件：

```
https://downloads.immortalwrt.org/
```

> **Cudy TR3000 用户**：选择 `ipq50xx` 平台，建议选带 `ext4` 或 `squashfs` 的 sysupgrade 包。

## 三、ImmortalWrt 旁路由网络配置

### 3.1 连接设备

用网线将 ImmortalWrt 的 **LAN 口**连接到主路由的 **LAN 口**（不是 WAN 口）。

> ⚠️ 旁路由模式下，ImmortalWrt 的 WAN 口**不需要接线**，所有流量走 LAN 口。

### 3.2 登录后台

ImmortalWrt 默认地址：`192.168.1.1`，用户名 `root`，密码默认为空或 `password`。

初次登录建议先修改密码：**系统 → 管理权 → 修改密码**。

### 3.3 配置 LAN 口 IP

进入 **网络 → 接口 → LAN → 编辑**，修改以下参数：

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 协议 | 静态地址 | 旁路由必须用静态 IP |
| IPv4 地址 | `192.168.1.2`（举例） | 需与主路由同网段，且不冲突 |
| IPv4 子网掩码 | `255.255.255.0` | 与主路由一致 |
| IPv4 网关 | `192.168.1.1` | 填主路由 IP |
| DNS 服务器 | `192.168.1.1` 或 `223.5.5.5` | 主路由 IP 或公共 DNS |

> **示例**：主路由为 `192.168.1.1`，旁路由 LAN 设为 `192.168.1.2`。

### 3.4 关闭 DHCP

旁路由**不负责分配 IP**，DHCP 由主路由统一管理。

进入 **网络 → 接口 → LAN → 编辑 → DHCP 服务器**：

- 勾选 **忽略此接口**（即关闭 DHCP）

保存并应用。

### 3.5 关闭 IPv6（可选但推荐）

IPv6 可能导致流量绕过旁路由，建议关闭：

**网络 → 接口**，找到 `LAN` 接口 → 编辑 → **IPv6 设置** → 全部关闭或删除 IPv6 相关接口（如 `lan6`）。

## 四、防火墙设置

### 4.1 开启 IP 伪装（Masquerade）

进入 **网络 → 防火墙 → 区域**，找到 `lan` 区域，确认：

- **IP 动态伪装**：✅ 勾选
- **MSS 钳制**：✅ 勾选（可选，防止大包丢失）

### 4.2 转发规则

同一页面，确保：

- `lan → lan` 转发：**接受**

保存并应用。

---

## 五、客户端指向旁路由

有两种方式让设备走旁路由。

### 方式一：单设备手动设置（精准控制）

在需要走旁路由的设备上，手动修改网络配置：

- **IP 地址**：自定或 DHCP 获取（维持不变）
- **子网掩码**：`255.255.255.0`
- **默认网关**：`192.168.1.2`（旁路由 IP）
- **DNS**：`192.168.1.2` 或 `223.5.5.5`

### 方式二：主路由 DHCP 下发网关（全局生效）

在**主路由**的 DHCP 设置中，将 **默认网关** 改为旁路由 IP（`192.168.1.2`），所有连接主路由的设备都会自动走旁路由。

> ⚠️ 此方式影响全部设备，旁路由故障时全网无法上网，请谨慎选择。

## 六、验证连通性

### 6.1 检查路由是否正常

在客户端 ping 主路由和外网：

```bash
ping 192.168.1.1   # 能通：主路由可达
ping 8.8.8.8       # 能通：外网可达
tracert 8.8.8.8    # 查看是否经过旁路由
```

`tracert` 结果中第一跳应为旁路由 IP（`192.168.1.2`），第二跳为主路由（`192.168.1.1`）。

### 6.2 检查 DNS 解析

```bash
nslookup baidu.com
```

能正常解析则 DNS 配置正确。

## 七、常见问题排查

### ❌ 旁路由能 ping 通主路由，但无法上网

**检查项：**
- 防火墙 `lan` 区域是否开启了 **IP 伪装**
- 旁路由网关是否正确填写主路由 IP
- 确认主路由允许旁路由子设备上网（通常默认允许）

### ❌ DNS 解析失败

**检查项：**
- LAN 接口 DNS 是否填写正确
- homeproxy DNS 设置是否与路由规则冲突
- 尝试临时改为 `223.5.5.5`（阿里）或 `119.29.29.29`（腾讯）

### ❌ 部分设备不走旁路由

**原因：** 设备获取到的网关仍是主路由。

**解决：** 手动修改该设备的网关为旁路由 IP，或在主路由 DHCP 中统一下发。

### ❌ IPv6 流量绕过代理

**解决：** 参考第 3.5 节，在 ImmortalWrt 上禁用 IPv6，并在客户端也禁用 IPv6。

### ❌ 旁路由重启后 IP 冲突

**原因：** 主路由 DHCP 动态分配了旁路由 IP 给其他设备。

**解决：** 在主路由 DHCP 中为旁路由 MAC 地址绑定静态 IP。

## 九、网络拓扑示意

```
[互联网]
    │
[主路由 192.168.1.1]
    │ LAN
    ├─── [旁路由 192.168.1.2 (ImmortalWrt)]
    │         ↑ 客户端默认网关指向此处
    ├─── [PC / 手机 / 其他设备]
    └─── [其他设备（不走旁路由）]
```

---

> 📌 **小结**：旁路由配置核心只有三点——**静态 IP + 关 DHCP + 开 IP 伪装**，其余均为可选增强功能。配置完成后建议重启旁路由设备，再验证连通性。
